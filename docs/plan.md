# 🚀 [JetPipe] 제품 및 개발 계획서 (Product Plan)

## 📌 1. 제품 개요 (Product Vision)

- **서비스명**: JetPipe (제트파이프)
- **의미**: 서버와 서버 사이를 제트기처럼 초고속으로 연결하는 메모리 스트리밍 파이프라인
- **한 줄 요약**: 내 컴퓨터 하드디스크를 거치지 않고, 서버 간 대용량 파일을 인메모리 스트리밍으로 다이렉트 교차 전송하는 맥/윈도우 공용 초경량 데스크톱 SFTP 클라이언트

### 핵심 타겟

- 맥북을 사용하며 서버 간 파일 이동 시 `scp` 타임아웃과 복잡한 경로 타이핑에 지친 백엔드·데브옵스·AI 엔지니어
- 모바엑스텀(MobaXterm)의 '세션 12개 제한'과 90년대 아재 UI에 지쳐 대안을 찾는 윈도우 엔지니어

---

## 🛠️ 2. 기술 스택 (Tech Stack)

### 프론트엔드 (UI)
- **React.js + TailwindCSS** (Shadcn/ui 조합)
- 노션·피그마 스타일의 극도의 미니멀하고 세련된 다크모드 대시보드 구현

### 백엔드 (App Core)
- **Tauri (Rust)**
- **선정 이유**:
  - 일렉트론 대비 10분의 1 수준인 10MB 미만의 초경량 앱 용량 구현
  - 하드웨어 성능을 100% 뽑아내는 Rust 언어 특유의 소켓 통신 속도 활용

### 핵심 라이브러리
- **ssh2-rs**: Rust용 고성능 SSH/SFTP 제어 라이브러리 (libssh2 바인딩 — GitHub/AWS CLI 등에서도 사용하는 검증된 엔진)
- **keyring**: OS 네이티브 자격증명 저장소 (macOS Keychain / Windows Credential Manager / Linux Secret Service) — 시크릿은 앱이 직접 저장하지 않고 OS 금고에 위임

## 📅 3. JetPipe 3단계 출시 로드맵

### 🟩 1단계: JetPipe MVP (최소 기능 제품) 개발 (1~2달 소요)

불필요한 잡기능은 전부 배제하고, 사용자가 겪는 "불편함"을 해결하는 핵심 기능 한 가지만 완벽하게 구현합니다.

**주요 구현 기능**:

1. **무제한 세션 보관함**: 서버 접속 정보(IP, Key 등)를 유저 로컬에 암호화하여 개수 제한 없이 무제한 무료 저장
2. **듀얼 SFTP 패널 UI**: 화면을 좌/우로 쪼개서 서버 A와 서버 B의 폴더 구조를 시각적으로 보여주는 UI
3. **Jet-Stream 드래그 앤 드롭**: 서버 A의 파일을 마우스로 드래그해서 서버 B로 던지면, 내부 Rust 엔진이 로컬 RAM 버퍼를 통해 실시간으로 `stream.pipe` 해주는 핵심 전송 로직
4. **리스크 방어 전략**: 배포 시 MIT 라이선스를 명시하여 데이터 유실 및 전송 실패에 대한 법적 면책 장치 마련

## 📊 4. 안정성 및 비용 리스크 방어 대책

| 예상되는 위험 (Risk) | JetPipe의 방어책 (Countermeasure) |
|---------------------|----------------------------------|
| 대기업 서버 데이터 유실 시 소송 위험 | MIT 라이선스 면책 조항을 약관 및 깃허브 메인에 필수 명시하여 법적 책임 차단 |
| 유저 증가로 인한 서버 비용 폭탄 | **0(Zero) 서버 구조**. 모든 데이터 전송은 유저 본인의 RAM과 인터넷을 쓰므로 유저가 100만 명이 되어도 서버 비용 0원 |
| 보안 문제 (서버 키 탈취 우려) | 시크릿(패스워드/passphrase)을 **OS 네이티브 키체인**(Apple Keychain / Windows Credential Manager)에 위임 저장. 앱이 평문 파일로 보관하지 않음 |
| 서버 간 방화벽으로 인한 통신 차단 | 두 서버가 서로를 못 찾아도, 유저 PC가 양쪽을 SSH 터널링으로 중계하므로 무조건 전송 성공 |
| 전송 중 네트워크 단절 → 반쪽 파일 잔존 | ssh2의 블로킹 소켓 + TCP 흐름 제어로 백프레셔 자동, 실패/취소 감지 시 목적지 미완성 파일을 즉시 `unlink`로 제거 |
