1. 🔑 키 저장의 무결성: 'OS 순정 금고' 빌려 쓰기 (Keytar)
"유저의 마스터 키나 패스워드를 내 앱이 파일로 저장했다가 유출되면 어쩌지?"라는 두려움을 완벽하게 해결하는 기술입니다. 내 앱이 직접 암호화 코드를 짜서 하드디스크에 저장하면 보안 취약점이 생길 수 있습니다.

해결책: 맥북에는 '키체인(Keychain)', 윈도우에는 '자격 증명 관리자(Credential Manager)'라는 OS 자체 내장 최고 보안 금고가 있습니다. 애플과 마이크로소프트가 수조 원을 들여 해킹이 불가능하게 만든 금고죠.

구현 방법: 우리 앱은 유저의 비밀번호를 직접 보관하지 않고, Rust/NodeJS의 Keytar 같은 라이브러리를 통해 "애플 키체인 형님, 이 비밀번호 좀 대신 보관해 주세요" 하고 맡겨버립니다.

결과: 우리 앱의 보안 수준이 곧바로 애플/MS 수준으로 격상됩니다. 내 앱이 해킹당해도 금고는 OS가 잠그고 있으므로 안전합니다.

2. 🌊 전송의 무결성: 데이터 유실을 막는 '백프레셔(Backpressure)' 제어
"10GB짜리 보내다가 와이파이 끊겨서 파일 깨지면 어쩌지?"라는 성능적 문제를 해결하는 기술입니다.

해결책: 리눅스/NodeJS/Rust 생태계의 모든 스트림(Stream) 통신에는 데이터가 잘 도착했는지 패킷 단위로 체크하는 TCP 핸드셰이킹과 배압(Backpressure) 제어 기술이 기본 탑재되어 있습니다.

원리: 서버 A에서 데이터를 읽어오는 속도가 100인데, 서버 B로 들어가는 속도가 50밖에 안 되면 중간에 50만큼 데이터가 밀려 앱이 터지거나 파일이 깨집니다. 이때 시스템이 알아서 "잠깐! 서버 B가 바쁘니까 서버 A 너 잠시 읽기 멈춰!"라고 브레이크를 밟아줍니다.

결과: 와이파이가 끊기면 에러 이벤트를 즉시 감지하여 전송을 안전하게 중단(Abort)시키고, 목적지 서버에 "이 파일은 비정상적으로 종료되었으니 파기해라" 혹은 "여기서부터 이어받아라"라는 명령을 내릴 수 있습니다. 파일이 껍데기만 전송되는 일은 발생하지 않습니다.

3. 🧩 오픈소스 라이브러리의 힘: ssh2 엔진 그대로 쓰기
우리가 SSH 통신 규약을 직접 코딩하는 게 아닙니다. 이미 전 세계 수백만 개의 인프라 시스템에서 쓰고 있는 검증된 오픈소스 엔진인 libssh2(또는 Rust의 ssh2-rs)를 가져와 씁니다.

깃허브, AWS CLI, 마이크로소프트조차도 자기들이 직접 SSH 코드를 짜지 않고 이 검증된 오픈소스 라이브러리를 가져다 씁니다. 수천 명의 천재 개발자들이 매일 버그와 보안 취약점을 감시하고 패치하고 있거든요.

우리가 할 일은 이 단단한 엔진 위에 "서버 A의 읽기 스트림과 서버 B의 쓰기 스트림을 파이프(.pipe())로 연결해라"라는 스위치 역할의 코드만 예쁘게 얹는 것입니다.
